## 交付

### W3 `quant-bit-curve`（混合精度 Pareto 位宽-精度曲线）

**定位**：与 W2（固定位宽对比 PTQ 算法）互补——W3 在精度约束下对比**位宽/格式**选择（INT8/W4A8/INT4/MX4/MX8，全 mxint 基），找 Pareto 前沿「最低平均位宽下最高精度」。

- **底层**：`ts_quant.search_mix_precision(strategy="m0_pareto", mode=explore/constrained_select/minimize_bit_under_accuracy)`。
- **agent**：`bit-curve-searcher`（folder-agent，单节点）+ `run_bit_curve.py`（~510 行）。
- **可视化**：line（Pareto 位宽-精度曲线，主图）+ bar（选中候选格式分布）+ table（前沿候选）。
- **bake**：`final.layer_configs` 每层 dict → `QConfig.from_dict` → `quantize_model(qconfig_dict=...)` → `best_mixed_model.pt`。
- **格式→QConfig 表**：复刻 SDK `_default_format_aliases`（INT8/W4A8/INT4/MX4/MX8 → QConfig，granularity 跟输入）。脚本侧维护此映射因 `candidate_format_space` 只吃 QConfig 对象不吃字符串别名（探针实证 `TypeError: Unsupported candidate type: <class 'str'>`）。

### W4 `quant-qat`（量化感知训练 + CAGE 后校正）

**定位**：W1–W3 都是训练后量化（PTQ），W4 是训练感知量化（QAT）——fake-quant 模型掉精度后短训恢复（+ CAGE 后校正 `W←W−lr·λ·(W−Q(W))`）。

- **底层**：`prepare_trainable_fakequant_model(scheme=rtn|duquantpp)` + `prepare_trainable_qat(cage=auto)` + 训练 loop（`opt.step()` 后 `qat.step()`）。
- **agent**：`qat-trainer`（folder-agent，单节点）+ `run_qat.py`（~480 行）。
- **对比轴**：scheme（rtn vs duquantpp，both 模式双方案对比），绘图 = per-step 收敛曲线 + 训练前/后精度恢复。
- **训练 loss**：teacher-student mse（label-free distillation-style QAT，`loss=mse(q(batch), fp(batch).detach())`），与默认 eval 口径一致。
- **可视化**：line（per-step 收敛，每 scheme 一条）+ bar（前/后精度）+ table（方案对比）。
- **bake**：最佳 q_model `state_dict()` → `best_qat_model.pt`。

### 探针实证的 API 事实（de-risk 600 行脚本）

写脚本前在 demo_target ViT-Tiny 上探针，钉死：

- **W3**：`q_layers` = dict（`TSQuantizer.prepare().q_layers`）；`candidate_format_space` 必须是 QConfig 列表（不是字符串）；`report` 顶层键 + `frontier.points[i]` 字段名（`candidate_id/primary_metric/selection_bit_score/avg_wa_bit_proxy/format_counts/accuracy_loss`）+ `final` 字段（`score/selected_candidate_id/selection_bit_score/format_counts/layer_configs`）+ 落盘文件（`bit_trend.json`/`frontier.json`/`best_under_constraints.json`）。
- **W4**：`prepare_trainable_fakequant_model` 返回 `(q_model, report)`；`prepare_trainable_qat` 返 `TrainableQAT`（`qat.step()` 每 optimizer step 调一次，`has_internal_cage`/`cage_start_step`）；`DuQuantPPConfig` 默认字段。3 步训练 loop 实测跑通（loss ~1e-4）。
- **W4 两处 duquantpp 约束**（探针发现并修）：① `DuQuantPPConfig` 要求显式 `target_patterns`（默认空 → raise；改 `(".*",)` + module_types=linear 预过滤 = 全 Linear 层，与 rtn 公平对比）；② `block_size` 必须与 qconfig.block_size 一致（默认 32 vs qconfig 16 → raise；改 `block_size=qconfig.block_size`）。
