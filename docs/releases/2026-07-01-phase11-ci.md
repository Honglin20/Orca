## 背景

phase 11 wave 1 起步，**P0.1 CI** 是最低风险、无 blocker 的第一项（SPEC §10.1 验收第 1 条「`git push` → CI 自动跑测试，绿」）。生产就绪硬要求，目的：防回归、固定基线。

仓库此前**无 `.github/` 目录**，本任务从零建两个 GitHub Actions workflow。

## 改动

### 新增文件（仅 2 个 YAML，surgical）

| 文件 | 作用 |
|---|---|
| `.github/workflows/test.yml` | **CI gate**：push(master)/PR(master) → matrix Python 3.10/3.11/3.12 → `uv sync` → `uv run pytest tests/ -m "not integration"` |
| `.github/workflows/integration.yml` | **opt-in 真 claude E2E**：PR comment 含 `/integration` → guard 校验（PR-only + write 权限 + 非 fork PR）→ checkout PR HEAD → 装 claude CLI → `uv run pytest tests/iface/cli/test_integration.py -m integration`（注入 `ANTHROPIC_API_KEY`） |

### 关键设计决策

1. **gate 永不跑 integration**：integration 测试需真 spawn claude + API key（`tests/iface/cli/test_integration.py` 整模块 `pytestmark = pytest.mark.integration`），gate 用 `-m "not integration"` 排除，确保 PR 每次推送都跑得起、跑得快。matrix 3.10/3.11/3.12 覆盖 `requires-python = ">=3.10"` 全区间。
2. **integration 是 opt-in slash command**：`issue_comment` + `contains(comment.body, '/integration')` 触发，避免每次 PR 自动烧 API key。`concurrency` 按 PR 号取消旧 run。
3. **fail-loud 三道防线**（呼应 CLAUDE.md Rule 12）：
   - **fork PR 拒绝**：guard job 校验 `pr.head.repo.fork === false`，防 fork HEAD 代码（外部控制）跑测试 + 烧 key。
   - **write 权限校验**：`getCollaboratorPermissionLevel` 限 admin/maintain/write，防陌生人烧 key。
   - **API key 非空前置校验**：integration job 跑 pytest 前先 `[ -z "$ANTHROPIC_API_KEY" ] && exit 1`，否则 `_has_claude()` 会静默全 skip → CI 假绿。
4. **action 版本 pin major**：`actions/checkout@v4`、`actions/setup-python@v5`、`astral-sh/setup-uv@v3`、`actions/github-script@v7`（任务要求，patch/minor 自动跟进）。
5. **timeout-minutes**：gate 15 分钟、integration 30 分钟（真 claude spawn 慢），防挂死浪费额度。
6. **trigger 用 `contains` 而非 `==`**：review [major-1] 指出严格相等会让带尾随空格/换行的 `/integration` comment 静默不触发（GitHub 移动端/某些编辑器会附空白），改 `contains` 贴合 SPEC「含」措辞。
