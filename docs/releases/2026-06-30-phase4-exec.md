## 交付清单

### 步骤 A：契约层（后端无关地基）
- `orca/exec/interface.py` —— `Executor` ABC：`async exec(node, ctx) -> AsyncIterator[Event]`；事件生命周期契约（node_started → 流式 → node_completed | node_failed）。abstract 方法体用 `if False: yield` 满足 AsyncIterator 类型注解语法。
- `orca/exec/context.py` —— `RunContext`（frozen dataclass）：inputs / outputs / run_id；frozen 保证执行快照不可变（node 间由 orchestrator 构造新实例累加上游输出）。
- `orca/exec/error.py` —— `ExecError(phase, message, error_type=None)` + `phase_to_error_type(phase)`；6 类 phase→error_type 映射表（timeout/spawn/stream/result_parse/schema/render）；未知 phase 抛 ValueError（映射表漏补是 bug，fail loud）。
- `orca/exec/factory.py` —— `make_executor(node) -> Executor`：按 `isinstance(node, ...)` 分派 AgentNode→ClaudeExecutor / ScriptNode→ScriptExecutor / SetNode→SetExecutor / ForeachNode→NotImplementedError（归 phase 5）；非 4 种 kind 抛 TypeError（fallback fail loud）。各 import 惰性（避免 `__init__` import 时拉起全链）。
- `orca/exec/__init__.py` —— 导出 Executor / make_executor / RunContext / ExecError / phase_to_error_type；ClaudeExecutor/ScriptExecutor/SetExecutor 经 `__getattr__` 惰性解析（SPEC §7.1 要求可 `from orca.exec import ClaudeExecutor`）。

### 步骤 B：共享基础设施
- `orca/exec/render.py` —— Jinja2 共享渲染层：`render_template` / `render_command` / `render_prompt`；`StrictUndefined`（未定义变量 fail loud → ExecError(phase=render)）；命名空间把 ctx.inputs + ctx.outputs（每个 node 的 output 以 node 名为顶层 key）摊开；`render_prompt(node=None)` 走 `agents/<name>.md` 约定加载。
- `orca/exec/runner.py` —— `CLIRunner`（通用 asyncio 子进程基础设施）：`SpawnConfig` + `CliRunResult` + `CLIRunner`；spawn（`create_subprocess_exec` + shlex.split cli_path）→ stdin pump（写 UTF-8 + drain + close + wait_closed）→ readline 循环（按 `\n` 分割，空行跳过）→ result 行检测（`json.loads` 后 `type=="result"` 回调 `on_result`，含 is_error 透传）→ 超时（每行 readline 包 wait_for，超时 SIGTERM → 10s grace → SIGKILL，kill 单进程）→ stderr 累积（上限 64KB，留末尾）。非 JSON 心跳行 debug log + 跳过（SPEC §6 json_decode 例外，不 fail loud）。

### 步骤 C：ClaudeExecutor 核心
- `orca/profiles/translators/claude.py` —— **真 `claude_translator`**（决策 1：归属 profiles，见下）纯函数：按顶层 type 分派（stream_event→text/thinking 增量片段 / input_json_delta 不翻译 / assistant tool_use→tool_call / user tool_result / result→agent_usage / system api_retry→error warning）；seq=0 占位 + timestamp=time.time() 真实（决策 2）；不抛（未知/非 JSON 返回 `[]`）；agent_usage 只发一次；result 文本不经 translator（走 on_result）。
- `orca/profiles/translators/__init__.py` —— 导出 claude_translator。
- `orca/profiles/builtin/claude.py` —— PROFILE 的 translator 从 `_dummy_translator` 换成真 `claude_translator`（删 dummy）；result_extractor 保 dummy（决策 3）。
- `orca/exec/claude/result_extractor.py` —— `extract_json_text`（整段 JSON / ```json fence``` / 第一个平衡 `{...}``/`[...]` 块，处理嵌套+字符串内括号+转义）+ `extract_and_validate(text, schema)`（schema=None 返回原文；非 None → 提取 + jsonschema 校验，失败 ExecError(phase=schema)）。
- `orca/exec/claude/executor.py` —— `ClaudeExecutor(profile)`：exec 12 步流程（session_id=uuid4 → node_started → render_prompt → build_spawn_config → CLIRunner + on_result holder → 逐行喂 profile.translator + 富化 node 字段 → 有序互斥判定 §2.4 → extract_and_validate → node_completed 带输出+elapsed+usage / except ExecError → node_failed + error 双发）。argv 动态拼（--model / --allowed-tools 单 flag+空格 join）；env overlay 透传 profile 声明前缀。
- `tests/profiles/fixtures/sample_with_bash.jsonl` —— 从 AgentHarness `_fixtures/` 只读拷贝（42 行真实 stream-json，claude_code_version 2.1.150）。

### 步骤 D：script / set
- `orca/exec/script.py` —— `ScriptExecutor`：subprocess_shell + wait_for(timeout) → {stdout, stderr, exit_code}；parse_json 失败→output.json=None（降级不阻断）；**非零退出码不 fail loud**（业务语义，由路由判断）；**timeout 必须 fail loud**（node_failed phase=timeout）；spawn OSError→phase=spawn。
- `orca/exec/set_node.py` —— `SetExecutor`：逐 key Jinja2 求值 → output dict；任一渲染失败→ExecError(phase=render)→node_failed；纯计算无子进程。

### 步骤 E：集成 + 端到端
- `orca/exec/factory.py` —— 真分派（见步骤 A）。
- `tests/exec/conftest.py` —— autouse `_reset_profiles_registry` + `full_stream_lines` fixture（共享，pytest 自动发现）。
- `pyproject.toml` —— 加 `jsonschema>=4.0` 运行时依赖（result_extractor 跑时即用）；注册 `integration` pytest marker。

## 3 条架构决策覆盖（覆盖 SPEC 个别措辞，理由充分，已落地）

### 决策 1：translator 归 profiles 层，不归 exec（覆盖 SPEC §4.1 的文件结构图）

SPEC §4.1 把 `claude_translator` 画在 `orca/exec/claude/translator.py`。本实现修正为 `orca/profiles/translators/claude.py`（**逻辑归属 profiles**）。

**理由**：translator 是 per-backend 协议知识（每个 backend 一个），profile 是「这个 backend 的完整描述」单元；phase 3 已把 translator 定义为 `CliProfile.translator` 字段（`profiles/base.py`）。放 profiles 才能满足：①内聚（backend 知识不撕裂）；②OCP（加 backend=丢一个 profile 文件 + 一个 translator 文件，零 factory/exec 改动）；③依赖铁律（profiles 不能 import exec；translator 只依赖 schema，profiles 持有它零新增依赖）。

**落地**：
- `orca/profiles/translators/claude.py` 定义真 `claude_translator`；`PROFILE.translator=claude_translator`。
- `ClaudeExecutor` 用 `self.profile.translator(line, session_id)`。
- translator 测试在 `tests/profiles/test_claude_translator.py`（**不是** SPEC §7.11 写的 `tests/exec/claude/test_translator.py`）。
- **铁律验证**：`grep -rn "import orca.exec\|from orca.exec" orca/profiles/` 零匹配。

### 决策 2：Event 的 seq 占位（覆盖 SPEC §4.3 的 Event 构造细节）

`Event.seq` 是全局单调递增、由 `Tape.append` 写时分配（见 `events/bus.py` 的 emit）。phase 4 executor **不写 tape**（铁律 2），无法知道全局 seq。

**约定**：executor/translator 产出 Event 用 `seq=0` 占位 + `timestamp=time.time()` 真实；**phase 5 orchestrator 在 `tape.append` 时重分配 seq**——与 phase 3 emit 语义完全一致。代码注释说明「seq=0 占位，orchestrator 重分配」。phase 4 测试只断言 type/data/node/session_id，不断言全局 seq。

（实现早期曾用「单次 exec 内局部递增计数器」，review 后为一致性统一改成 `seq=0`——与 translator / script / set 的占位约定对齐。）

### 决策 3：result_extractor 拆两半

SPEC §4.3 里 `extract_and_validate` 是**通用能力**（JSON 提取 fence/balanced-block + jsonschema 校验，backend 无关）。留在 `orca/exec/claude/result_extractor.py`（SPEC §4.1 这条不变），ClaudeExecutor 直接 import。`profile.result_extractor` 字段保 dummy（ClaudeExecutor 不经它；claude 的 per-backend 知识经 CLIRunner 的 `on_result` 钩子处理，避免引入新耦合——Simplicity First）。

## 与 SPEC 草图的细微偏离（透明列出，已论证）

1. **CLIRunner.on_result 回调签名 4 参数**（SPEC §4.4 草图写 3 参数 `(raw, usage, cost)`）。本实现加第 4 参数 `is_error: bool`。**理由**：SPEC §2.4 第 3 项要求检测 `result.is_error==true`（phase=stream），而 executor 没有其他通道看到该 flag（translator 是纯函数，is_error 行返回 `[]`）。on_result 透传 is_error 是必要桥接，runner.py:52-56 有注释说明。
2. **ClaudeExecutor 富化 translator 事件的 node 字段**（SPEC §4.3 第 7 步伪代码省略）。translator 纯函数只收 `(line, session_id)`，不知 node 名；但 SPEC §4.2 要求「所有事件顶层带 node + session_id」。故 executor 在 yield 前用 `ev.model_copy(update={"node": node.name})` 补 node（executor.py:121）。
3. **factory 一次写真分派**（计划 A.4 写「骨架阶段全 raise NotImplementedError」）。本实现单 session 完成 A→E，直接写真分派（避免废弃骨架代码）；契约测试因此不测骨架 NotImplementedError，改由 `test_factory.py`（E.4）测真分派。
4. **asyncio.run 而非 pytest-asyncio**：遵循本仓库既有约定（`tests/events/test_bus.py` 明写「本仓库 dev 依赖仅 pytest，异步统一 asyncio.run」）。所有 exec 异步测试用 `asyncio.run`，无 `@pytest.mark.asyncio`。
5. **CLIRunner 超时是「两行之间」间隔**（每行 readline 各自包 wait_for），非总墙钟。phase 4 ClaudeExecutor 传 timeout=None（不限制），phase 5 若需总墙钟可在 exec 外层包 wait_for。runner.py 有注释说明。

## Review 反馈与修复
