| 步骤 | 模块 | 产出文件 | 测试文件 | 依赖 |

## 2. 步骤 B —— 共享基础设施

### B.1 `orca/exec/render.py`（Jinja2 渲染，三种 executor 共享）
- `render_template(tpl: str, ctx: RunContext) -> str`：Jinja2 渲染
  - 注入变量：`inputs`（ctx.inputs）+ **每个 node 的输出以 node 名为 key**（`ctx.outputs` 展开成顶层变量，`optimizer.output.structure` → `{{ optimizer.output.structure }}`）
  - 失败 raise `ExecError(phase="render", error_type="RenderError", message=...)`
- `render_prompt(node, ctx) -> str`：组装 prompt
  - `node.prompt` 非空 → 内联（渲染）
  - `node.prompt is None` → 从 `agents/<node.name>.md` 加载（与 compile 约定一致）；文件不存在 raise ExecError(phase="render")
- `render_command(command, ctx) -> str`：ScriptNode.command 渲染（同 render_template）

### B.2 `orca/exec/runner.py`（CLIRunner，通用 asyncio 子进程）
- `@dataclass SpawnConfig`：cli_path / flags / extra_args / mcp_flag_args / prompt / prompt_channel / env_overlay / timeout
- `@dataclass CliRunResult`：exit_code / stderr / timed_out / elapsed
- `class CLIRunner`：
  - `__init__(cfg, on_result=None)`：`on_result: Callable[[str, dict, float], None]`（result 文本/usage/cost 回调）
  - `async def stream(self) -> AsyncIterator[str]`：spawn（`create_subprocess_exec`，shlex.split cli_path）→ stdin pump（写+close）→ readline 循环 yield 行 → 检测 result 行回调 on_result
  - 属性 `timed_out` / `exit_code` / `elapsed`
  - 超时：`asyncio.wait_for(proc.wait(), timeout)` 超时 → SIGTERM → 等 10s → SIGKILL；单进程 kill（非 killpg）

### B.3 验收 — `tests/exec/test_render.py`
- [ ] `render_template("{{ inputs.x }}", ctx)` 正确取值
- [ ] `render_template("{{ optimizer.output.structure }}", ctx)` 从 ctx.outputs["optimizer"] 取（嵌套）
- [ ] 渲染未定义变量 → raise `ExecError(phase="render")`
- [ ] `render_prompt` 内联 prompt → 渲染后返回
- [ ] `render_prompt` None + agents/<name>.md 存在 → 加载内容（用 tmp_path 建假 md 测）
- [ ] `render_prompt` None + 文件不存在 → raise ExecError

### B.4 验收 — `tests/exec/test_runner.py`（**假子进程，不 spawn claude**）
- [ ] spawn `sys.executable -c "print('line1\nline2')"` → stream() yield "line1"/"line2"

### C.2 `orca/exec/claude/translator.py`（纯函数，核心可测点）
- `def claude_translator(line: str, session_id: str) -> list[Event]`
- `json.loads(line)`，按顶层 `type` 分派（SPEC §3.1 映射表）：

| 输入 | 输出 |
|---|---|
| `stream_event` + `text_delta` | `[Event(type="agent_message", data={text}, session_id)]` |
| `stream_event` + `thinking_delta` | `[Event(type="agent_thinking", data={text}, session_id)]` |
| `stream_event` + `input_json_delta` | `[]` |
| `assistant` 含 `tool_use` | `[Event(type="agent_tool_call", data={tool,args,tool_call_id}, session_id)]` |
| `user` 含 `tool_result` | `[Event(type="agent_tool_result", data={tool_call_id,result}, session_id)]` |
| `result` + `subtype=success` | `[Event(type="agent_usage", data={input,output,cache,cost}, session_id)]` |
| `result` + `is_error=true` | `[]`（executor 层处理） |
| `system` + `api_retry` | `[Event(type="error", data={...,retry_count,wait_seconds}, session_id)]`（warning 级） |
| 未知 type | `[]`（debug log，不抛） |

- **纯函数**：无 self、无副作用、无 I/O、无全局状态

### C.3 `orca/exec/claude/result_extractor.py`（纯函数）
- `def extract_json_text(text: str) -> Any`：按优先级
  - 整个 text 是合法 JSON → 直接返回
  - ```json fence``` / ```fence``` → 取围栏内
  - 第一个平衡 `{...}` 或 `[...]` 块（处理嵌套 + 字符串内括号 + 转义）
  - 都失败 → raise
- `def extract_and_validate(text: str, schema: dict | None) -> Any`：
  - `schema is None` → 返回原 text（自由文本）
  - 非 None → `extract_json_text` → 用 `jsonschema` 或简单 schema 校验 → 失败 raise `ExecError(phase="schema")`

### C.4 `orca/exec/claude/executor.py`（ClaudeExecutor）
- `class ClaudeExecutor(Executor)`：`__init__(profile: CliProfile)`
- `async def exec(self, node, ctx) -> AsyncIterator[Event]`：
  1. `session_id = uuid4().hex`
  2. `yield node_started(node, session_id)`
  3. `prompt = render_prompt(node, ctx)`
  4. `cfg = build_spawn_config(node, profile, prompt)`：argv 动态拼（`--model`/`--allowed-tools`/`--append-system-prompt`），flags 来自 profile
  5. `result_holder` + `on_result` 回调收集 result 文本/usage/cost
  6. `runner = CLIRunner(cfg, on_result)`
  7. `try: async for line in runner.stream(): for ev in profile.translator(line, session_id): yield ev`
  8. 判定（有序互斥，§2.4）：timed_out/exit_code/is_error/no_result → raise ExecError
  9. `output = extract_and_validate(result_text, node.output_schema)`
  10. `yield agent_usage`（若 on_result 收到 usage）
  11. `yield node_completed(node, session_id, {output, elapsed})`
  12. `except ExecError: yield node_failed + error`

## 4. 步骤 D —— script / set（确定 kind 执行）

### D.1 `orca/exec/script.py`（ScriptExecutor）
- `class ScriptExecutor(Executor)`
- `async def exec(self, node, ctx)`：
  1. `yield node_started`
  2. `cmd = render_command(node.command, ctx)`
  3. `proc = await create_subprocess_shell(cmd, stdout=PIPE, stderr=PIPE)`
  4. `stdout, stderr = await asyncio.wait_for(proc.communicate(), node.timeout)`（timeout None = 不限）
  5. timeout → raise ExecError(phase=timeout)
  6. `output = {stdout, stderr, exit_code}`；`node.parse_json` → `output["json"] = try_parse_json(stdout)`（失败 None，**不 fail loud**）
  7. **非零退出码不 fail loud**（业务语义，由路由判断）；正常 yield node_completed

### D.2 `orca/exec/set_node.py`（SetExecutor）
- `class SetExecutor(Executor)`
- `async def exec(self, node, ctx)`：
  1. `yield node_started`
  2. `output = {k: render_template(v, ctx) for k, v in node.values.items()}`
  3. Jinja2 渲染失败 → raise ExecError(phase=render) → node_failed
  4. `yield node_completed({output})`

## 6. 验收总则（SPEC §7.0，5 条铁律，每步自检）

1. **依赖单向**：exec→schema+events+profiles；**不依赖 run/compile**。判据：`grep -r "from orca.run\|from orca.compile" orca/exec/` 零匹配
2. **executor 不写 tape**：产出 `AsyncIterator[Event]`，写 tape 归 orchestrator。判据：`grep -r "from orca.events.bus\|EventBus\|Tape" orca/exec/` 零匹配（只允许 `from orca.schema import Event`）
3. **Translator 纯函数**：`(line, session_id)→list[Event]`，fixture 驱动测试，不 spawn。判据：translator 无 self/无 import 副作用模块
4. **fail loud**：6 类错误（timeout/spawn/stream/result_parse/schema/render）全 emit node_failed + error
5. **session_id 一致性**：单次 exec 内所有 Event.session_id 相同
