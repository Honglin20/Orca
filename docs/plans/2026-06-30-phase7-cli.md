## C3. OrcaApp（组装 + 事件消费 + 编排 worker）

### C3.1 `orca/iface/cli/app.py`
- `OrcaApp(App)`：`__init__(wf, inputs, task, max_iter)` 构造 bus + gate_handler
- `compose()`：Header + Horizontal(DagTree, Vertical(ActiveNode, LogStream)) + Footer
- `on_mount()`：subscribe 事件 + run_worker(_run_pipeline) + run_worker(_consume_events)
- `@work _run_pipeline()`：调 Orchestrator(...).run()
- `@work _consume_events()`：async for event → 分发（gate → push_screen_wait → resolve；其他 → widget 更新）
- `action_goto_gate()`：快捷键 g，跳到第一个 awaiting gate

### C3.2 关键实现点
- gate 双重身份处理：
  - `human_decision_requested` → push_screen_wait(GateModal) → resolve
  - `human_decision_resolved` → 若本壳 modal 还在，dismiss + 显示「已被 [source] 答」
- 事件 → widget 更新：node_started/completed/failed → DagTree 图标；agent_* → ActiveNode + LogStream
- 退出：orchestrator 跑完 → 显示结果面板 → q 退出 → exit code

### C3.3 验收（C3）— `tests/iface/cli/test_app.py`
- [ ] compose 返回 4 个 widget（Header/DagTree/ActiveNode/LogStream）
- [ ] 事件分发：注入 fake 事件流 → DagTree/LogStream 状态更新正确
- [ ] gate 流程：注入 human_decision_requested → GateModal 弹出（用 Textual pilot 测试）
- [ ] 编排 worker 启动（mock Orchestrator，验证 run 被调）
- [ ] 退出码：orchestrator 返回 completed → exit 0

### C3.4 测试用例骨架
```python
async def test_event_dispatch_to_widgets():
    """注入事件流，验证 widget 更新。"""
    app = OrcaApp(wf, inputs={}, task=None, max_iter=10)
    async with app.run_test() as pilot:
        # 注入 node_started
        app._dispatch_to_widgets(_make_event("node_started", node="a"))
        await pilot.pause()
        tree = app.query_one(DagTree)
        assert "✽" in _tree_node_label(tree, "a")  # running 图标

async def test_gate_modal_flow():
    """gate 弹窗 + resolve。"""
    app = OrcaApp(...)
    async with app.run_test() as pilot:
        app._dispatch_to_widgets(_make_event("human_decision_requested", data={...}))
        await pilot.pause()
```

## C4. Widgets + GateModal

### C4.1 `widgets/dag_tree.py`
- `DagTree(Tree)`：渲染所有 node（+ parallel 组为父节点）
- 状态图标：✓ done / ✽ running / ⏸ blocked / ! failed / ○ pending
- `update_node_status(name, status)` 方法
- parallel 组：父节点 + branches 子节点 + 进度计数

### C4.2 `widgets/active_node.py`
- `ActiveNode(Static)`：显示选中节点的行摘要 + 流式事件
- `set_active(name)`：切换选中
- `append_event(event)`：流式更新（agent_message/tool_call 等）

### C4.3 `widgets/log_stream.py`
- `LogStream(RichLog)`：流式日志，格式 `HH:MM:SS [session] <desc>`
- `append(event)`：格式化 + write

### C4.4 `widgets/header.py`
- `Header(Static)`：`Run #id · wf · model · done/total · ⏸ n awaiting`
- `update(stats)` 方法

### C4.5 `screens/gate_modal.py`
- `GateModal(ModalScreen[str])`：按 source 渲染
- tool_permission：工具名 + 参数 + 批准/拒绝/编辑按钮
- agent_ask：问题 + 选项/输入框
- `dismiss(answer)` 返回用户选择

### C4.6 验收（C4）— `tests/iface/cli/test_widgets.py` + `test_gate_modal.py`
- [ ] DagTree：5 种状态图标映射正确
