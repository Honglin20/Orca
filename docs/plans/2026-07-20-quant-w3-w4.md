## 1. W3 `quant-bit-curve` —— 位宽-精度曲线

### 1.1 定位（类比 W2 的「算法对比」轴）

W2 在**固定位宽**下对比预变换/求解算法（smooth/gptq/autoround/q2n）。W3 反过来：在**精度**约束下对比**位宽/格式**选择（INT8 / W4A8 / INT4 / MX4 / MX8，MX 家族即 mxint 基），找 Pareto 前沿「最低平均位宽下最高精度」。**对比轴 = 低比特格式**（全 mxint 基）。

底层 API：`ts_quant.search_mix_precision(strategy="m0_pareto", mode=...)` → 返回 `(best_configs, report)`，落盘 `bit_trend.json` / `frontier.json` / `best_under_constraints.json` / `report.json` / `frontier.png`。

### 1.2 输入契约（仿 W2，全 mxint 基）

| input | 必填 | 默认 | 说明 |
|---|---|---|---|
| `model_path` | ✅ | — | FP 模型入口 |
| `project_root` | ✅ | — | 项目根 |
| `calib_data_ref` | adv | `""` | calib loader dotted-path（空→假随机） |
| `eval_data_ref` | adv | `""` | eval loader（空→复用 calib） |
| `eval_fn_ref` | adv | `""` | 业务 eval_fn（空→teacher-student mse） |
| `mode` | default | `explore` | `explore`/`constrained_select`/`minimize_bit_under_accuracy` |
| `candidate_format_space` | default | `INT8,W4A8,INT4,MX4,MX8` | 对比格式集（逗号串） |
| `bit_objective` | default | `weight_activation_proxy` | bit 成本口径（绘图 x 轴） |
| `accuracy_tolerance` | default | `0.01` | absolute 精度损失容忍 |
| `avg_bit_budget` | adv | `""` | 硬 bit 上限（null） |
| `max_evals` | default | `32` | 主搜索预算（smoke 友好） |
| `granularity` | default | `per_tensor` | 量化粒度 |
| `output_dir` | default | `""` | 空→`llm_artifacts/<model_name>/` |
| `bake` | default | `true` | bake 选中最佳混合精度模型 |

### 1.3 单 agent（不复杂→单 agent，仿 W1/W2）

`bit-curve-searcher`：folder-agent，资源经 `ORCA_AGENT_RESOURCES` 锚定。
1. 确定 `<output_dir>`（空→推断 `llm_artifacts/<model_name>/`）。
2. 生成 `<output_dir>/adapter.py`：`load_model` / `get_calib_loader` / `get_eval_loader` / `forward_fn`（+ 业务路径 `get_eval_fn`/`get_metric_spec`，同 W2）。
3. 一条 bash 调 `run_bit_curve.py`（`source orca_env.sh` + `--env_file` 兜底，照抄 W2 反「拆 bash 丢 env」坑）。
4. 原样回显脚本 stdout JSON。

### 1.4 `run_bit_curve.py` 八步（确定性，~550 行，镜像 W2 骨架）

1. **import adapter** → FP teacher + calib + eval loaders + eval_fn（默认 `build_teacher_student_eval_fn`，metric_kind/higehr_is_better 由 eval_fn 路径决定，同 W2 `_resolve_eval`）。
2. **构造格式→QConfig 表**（复刻 SDK `_default_format_aliases`，全 mxint/int，granularity 跟输入）：
   - INT8=`int n8` / W4A8=`int n8 w4 a8` / INT4=`int n4` / MX4=`mx n4` / MX8=`mx n8`
3. **base qconfig + q_layers**：`TSQuantizer(fp, QConfig(method="int",n_bits=8,granularity)).prepare()` → `quantizer.q_layers`（dict）。
4. **`MixPrecisionSearchConfig`**（strategy=m0_pareto, mode, candidate_format_space=选中的 QConfig 列表, bit_objective, accuracy_tolerance={absolute,value}, avg_bit_budget, max_evals, output_dir）→ `search_mix_precision(fp, q_layers, eval_fn, metric_spec, search_config, return_report=True)`，try/except 兜全局失败（exit 3）。
5. **解析 report**：`report["frontier"]["points"]`（Pareto 点：bit_score + primary_metric）、`report["final"]`（score/selected_candidate_id/selection_bit_score/format_counts/layer_configs/avg_wa_bit_proxy）、`report["search_space_summary"]`/`["search_quality"]`（搜索质量信号）。聚合落 `report.json`（脚本自己的摘要，区别于 SDK 落盘的 report.json——放 output_dir 下 `bit_curve_summary.json` 避免覆盖）。
6. **bake**（bake=true）：`final.layer_configs` 每层 dict → `QConfig.from_dict` → `quantize_model(fp, qconfig_dict={name:qc})` → `torch.save(state_dict, best_mixed_model.pt)`。失败→空串 baked_model_path + stderr，不阻断（曲线是核心产出）。
7. **render_chart（容错不阻断）**：
   - **line**「Pareto 位宽-精度曲线」（x=`selection_bit_score`/avg_wa_bit_proxy，y=primary_metric，frontier.points 排序）——**主图**。
   - **bar**「选中候选格式分布」（format_counts：INT8/W4A8/INT4/MX4/MX8 层数）。
   - **table**「前沿候选」（bit_score | metric | format_counts | selection_reason）。
8. **stdout JSON**（对齐 output_schema，原样回显）。

### 1.5 output_schema

`output_dir, report_path, model_path, baked_model_path, best_config, best_metric, best_bit, candidates_evaluated, mode, metric_kind`
- `best_config`：选中候选标签（如 `cand_0003` + 主格式，或 `INT8×40+MX4×10`）。
- `best_metric`/`best_bit`：选中候选 primary_metric / selection_bit_score。
- `candidates_evaluated`：`report["eval_calls"]`。

## 2. W4 `quant-qat` —— 量化感知训练 + CAGE

### 2.1 定位（类比 W2 的「算法对比」轴）

W2 对比 PTQ 算法。W4 对比**训练态 fake-quant 方案**（`rtn` vs `duquantpp`）在 QAT（+CAGE 后校正 `W←W−lr·λ·(W−Q(W))`）下恢复精度的能力。故事：fake-quant 掉精度 → QAT(+CAGE) 恢复。**对比轴 = scheme**（rtn vs duquantpp），绘图 = 训练前/后精度 + per-step 收敛曲线。

底层 API：`prepare_trainable_fakequant_model(scheme=...)` + `prepare_trainable_qat(q_model, optimizer, total_steps, cage=...)` + 训练 loop（`opt.step()` 后 `qat.step()`）。

### 2.2 输入契约

| input | 必填 | 默认 | 说明 |
|---|---|---|---|
| `model_path` | ✅ | — | FP 模型入口 |
| `project_root` | ✅ | — | 项目根 |
| `calib_data_ref` | adv | `""` | calib loader（duquantpp scheme 需要；空→假随机） |
| `train_data_ref` | adv | `""` | 训练 loader（空→复用 calib 做最小 smoke QAT） |
| `eval_data_ref` | adv | `""` | eval loader（空→复用 train） |
| `eval_fn_ref` | adv | `""` | 业务 eval_fn（空→teacher-student mse） |
| `scheme` | default | `both` | `rtn`/`duquantpp`/`both`（both=对比两方案，仿 W2） |
| `bit_width` | default | `w8a8-mx` | QAT fake-quant 预设（w4a4-mx/w4a8-mx/w8a8-mx/w8a8-int/w4a16，复用 W2 `_BITWIDTH_PRESETS`） |
| `cage` | default | `auto` | `auto`/`true`/`false`（CAGE 后校正开关） |
| `total_steps` | default | `64` | QAT 训练步数（smoke 友好；真实 QAT 需更多） |
| `lr` | default | `1e-4` | Adam 学习率 |
| `output_dir` | default | `""` | 空→`llm_artifacts/<model_name>/` |
| `bake` | default | `true` | bake 最佳 q_model state_dict |

### 2.3 单 agent

`qat-trainer`：folder-agent。
1. 确定 `<output_dir>`。
2. 生成 `<output_dir>/adapter.py`：`load_model`（FP teacher）/ `get_calib_loader`（duquantpp 用）/ `get_train_loader`（空→复用 calib）/ `get_eval_loader`（空→复用 train）/ `forward_fn`（+ 业务 `get_eval_fn`/`get_metric_spec`）。
3. 一条 bash 调 `run_qat.py`（同 W2 env 兜底模式）。
4. 原样回显 JSON。

### 2.4 `run_qat.py` 流程（确定性，~550 行）

1. **import adapter** → FP teacher + calib + train + eval loaders + eval_fn（同 W2 `_resolve_eval`，默认 teacher-student mse；**训练 loss 也用 teacher-student mse**：`loss=mse(q_model(batch), fp_model(batch).detach())`，无需真实 label，distillation-style QAT）。
2. **scheme 集合**：`both`→{rtn,duquantpp}；单 scheme→{该}。
3. **逐 scheme**（try/except 隔离，单 scheme 失败不拖垮）：
   a. `prepare_trainable_fakequant_model(copy(fp), scheme=scheme, qconfig=qconfig, [calib_data/forward_fn/duquant_config=DuQuantPPConfig() for duquantpp])` → `(q_model, rep)`。
   b. **eval BEFORE**（fake-quant 基线，≈PTQ 精度）。
   c. `optimizer=Adam(q_model.parameters(), lr)` → `prepare_trainable_qat(q_model, optimizer, total_steps, cage=cage)`。
   d. **训练 loop**：`for step: batch=next(train_cycle); opt.zero_grad(); loss=teacher_mse; loss.backward(); opt.step(); qat.step(); 记录 step/metric/cage_lambda/phase`。
   e. **eval AFTER**（QAT 后精度）→ `recovery = after - before`。
4. **选 best**：`after_metric` 最优（higher_is_better 由 metric_kind 定；mse↓）。
5. **bake**（bake=true）：`torch.save(best_q_model.state_dict(), best_qat_model.pt)`。
6. **report.json**（per-scheme before/after/recovery/curve + best）。
7. **render_chart（容错）**：
   - **line**「QAT 收敛曲线」（x=step，y=metric，每 scheme 一条）——**主图**。
   - **bar**「QAT 前/后精度」（每 scheme before/after，示恢复）。
   - **table**「scheme | before | after | delta | steps | cage」。
8. **stdout JSON**。

### 2.5 output_schema

`output_dir, report_path, model_path, baked_model_path, best_scheme, best_metric, best_metric_before, recovery, schemes_evaluated, total_steps, cage, metric_kind`
