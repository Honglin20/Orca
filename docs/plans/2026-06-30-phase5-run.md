```python
class RouteError(Exception): ...   # NoRouteMatch

def resolve(routes: list[Route], output: Any, ctx: RunContext) -> str:
    """first-match-wins。无 when=兜底；全不匹配 raise RouteError。"""
    eval_ctx = {
        "output": output,
        "inputs": ctx.inputs,
        **ctx.outputs,   # 每个已完成 node 以名为 key
    }
    for route in routes:
        if route.when is None:
            return route.to
        if _eval_jinja2_bool(route.when, eval_ctx):
            return route.to
    raise RouteError(f"无 route 匹配（output={output}）")

    """Jinja2 渲染 when 表达式 → truthy 判定。失败 fail loud。"""
```

## R3. ExecutorAdapter（桥接 exec → events）

### R3.1 `orca/run/executor_adapter.py`
```python
async def execute_and_emit(executor, node, ctx, bus) -> Any:
    """拿 executor.exec() 的 AsyncIterator，逐个 bus.emit，返回 output。
    node_failed → raise ExecError。"""
    output = None
    async for event in executor.exec(node, ctx):
        bus.emit(event, session_id=event.session_id)
        if event.type == "node_completed":
            output = event.data["output"]
        elif event.type == "node_failed":
            raise ExecError(phase="node", message=...)
    return output
```

## R4. Orchestrator + Parallel + Foreach（核心）

### R4.1 `orca/run/orchestrator.py`（主循环）
```python
class Orchestrator:
    def __init__(self, wf, bus, inputs, task=None): ...
    async def run(self) -> RunState:
        emit workflow_started
        current = wf.entry
        iterations = 0
        try:
            while current != "$end":
                iterations += 1
                if iterations > max_iter: raise MaxIterationsError
                if current in parallel_groups: output = await _run_parallel(current)
                elif find_node(current).kind == "foreach": output = await _run_foreach(...)
                else: output = await _run_node(current)
                ctx.outputs[current] = output
                current = router.resolve(node.routes, output, ctx)
            final = evaluate_outputs(wf.outputs, ctx)
            emit workflow_completed(final)
        except (...) as e: emit workflow_failed
        return replay_state(bus.tape)
```

### R4.2 `orca/run/parallel.py`
```python
async def run_parallel_group(group, ctx, bus, wf) -> dict:
    async def run_one(branch_name):
        if branch_name in ctx.outputs: return ctx.outputs[branch_name]  # 幂等跳过
        node = wf.find(branch_name)
        executor = make_executor(node)
        return await execute_and_emit(executor, node, ctx, bus)
    results = await asyncio.gather(*[run_one(b) for b in group.branches], return_exceptions=True)
    return aggregate(results, group.failure_mode)  # {outputs,errors,count}
```

### R4.3 `orca/run/foreach.py`
```python
async def run_foreach(node, ctx, bus) -> dict:
    arr = eval_jinja2(node.source, ctx)
    sem = asyncio.Semaphore(node.max_concurrent)
    async def run_one(idx, item):
        async with sem:
            body_ctx = ctx.with_locals({node.item_var: item, node.index_var: idx})
            executor = make_executor(node.body)
            return await execute_and_emit(executor, node.body, body_ctx, bus)
    results = await asyncio.gather(*[run_one(i,x) for i,x in enumerate(arr)])
    return aggregate_foreach(results, node.failure_mode)  # {outputs:[...],errors:{},count}
```

### R5.3 demo workflow 矩阵（7 个，真 claude 端到端）

> **设计原则**：用 script/set 产生**确定性值**驱动路由，agent 用极简 prompt（如「回复 DONE」）。
> 路由依赖的值确定（不靠 claude），agent 即使输出不确定也能断言 workflow 跑通。

#### demo 1：`examples/demo_linear.yaml`（纯线性）
```yaml
name: demo_linear
entry: a
nodes:
  - {name: a, kind: script, command: "echo step_a", routes: [{to: b}]}
  - {name: b, kind: script, command: "echo step_b", routes: [{to: c}]}
  - {name: c, kind: script, command: "echo step_c", routes: [{to: $end}]}
outputs: {result: "{{ c.output.stdout }}"}
```
**覆盖**：线性推进、ctx 累积、outputs 求值。**全 script，零 token，最快**。

#### demo 2：`examples/demo_conditional.yaml`（条件分支）
```yaml
name: demo_conditional
entry: decide
nodes:
  - name: decide
    kind: set
    values: {path: "high"}     # 确定值
    routes:
      - {when: "output.path == 'high'", to: high_agent}
      - {to: low_agent}
  - {name: high_agent, kind: agent, prompt: "回复 HIGH", routes: [{to: $end}]}
  - {name: low_agent, kind: agent, prompt: "回复 LOW", routes: [{to: $end}]}
outputs: {taken: "{{ decide.output.path }}"}
```
**覆盖**：条件分支、兜底 route、agent 节点。

#### demo 3：`examples/demo_loop.yaml`（回环循环 + max_iter）
```yaml
name: demo_loop
entry: counter
nodes:
  - name: counter
    kind: set
    values: {n: "{{ inputs.start | default(0) | int + 1 }}"}   # 每轮 +1
    routes:
      - {when: "output.n >= 3", to: done}     # 计数到 3 停
      - {to: counter}                          # 否则回环
  - {name: done, kind: script, command: "echo done", routes: [{to: $end}]}
outputs: {final_count: "{{ counter.output.n }}"}
```
**覆盖**：回环循环、set 累积状态、循环终止条件。**用 set 计数，确定停在 3**。

#### demo 4：`examples/demo_foreach.yaml`（数组分批并行）
```yaml
name: demo_foreach
entry: maker
nodes:
  - name: maker
    kind: set
    values: {items: "[1,2,3]"}    # 确定数组
    routes: [{to: processor}]
  - name: processor
    kind: foreach
    source: "maker.output.items"
    item_var: item
    max_concurrent: 2
    body: {kind: script, command: "echo item {{ item }}"}
    failure_mode: continue_on_error
    routes: [{to: $end}]
outputs: {processed: "{{ processor.output.count }}"}
```
**覆盖**：foreach 分批、Semaphore 限并发、聚合 count。

#### demo 5：`examples/demo_mixed.yaml`（综合，简化 nas）
```yaml
name: demo_mixed
entry: prep
nodes:
  - {name: prep, kind: script, command: "echo data", routes: [{to: analyzer}]}
  - {name: analyzer, kind: agent, prompt: "分析 {{ prep.output.stdout }}，回复 OK", routes: [{to: judge}]}
  - name: judge
    kind: set
    values: {verdict: "pass"}
    routes:
      - {when: "output.verdict == 'pass'", to: reporter}
```

**覆盖**：script+agent+set 混合、回环、条件分支（综合 nas 的简化版）。

#### demo 6：`examples/demo_task.yaml`（task 位置参数注入）
