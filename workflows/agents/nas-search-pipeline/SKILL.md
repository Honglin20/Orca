# NAS Search Pipeline

This skill produces the project-specific execution scripts that drive a NAS pipeline after the supernet has been trained. Concretely, it covers block-level latency profiling against the trained supernet, multi-objective evolutionary architecture search over its search space, and an `AGENTS.md` scaffold that guides AI coding assistants through executing the generated pipeline, selecting architectures from the Pareto front, and generating and executing retrain/finetune scripts. The skill emits Python entry points and remote-runnable shell launchers for profiling and search, plus a documentation scaffold so users can run the pipeline and produce retrain scripts in later sessions with their preferred AI coding tool.

This skill picks up where the `supernet-train-script` skill leaves off — the upstream skills (`pytorch-model-optimizer` then `supernet-train-script`) produce the supernet, its inspector, the supernet training script, and a completed `supernet_summary.md` under `<output_dir>`. This skill is typically invoked in a **fresh session** that only sees those resulting artifacts, which is why both `<output_dir>` and `<user_project_root>` must be confirmed up front rather than inferred from the local environment. When generating execution scripts, read both the generated artifacts under `<output_dir>` and the original project under `<user_project_root>` for data pipeline, validation metric, batch structure, model-call signature, optimizer/scheduler, AMP, checkpoint convention, dummy input shapes, and any other training behavior.

Skill resource paths:

- Do NOT read files under `<skill_dir>/references/workflow-checklists/`. Those are exclusively consumed by the `workflow-verifier` subagent.

### NPU Compatibility: Disable `foreach` Optimizations

Huawei Ascend NPU does not support PyTorch's `foreach`-based multi-tensor optimization. In practice, the `foreach` parameter appears in two places in training code: optimizer constructors (`torch.optim.*`) and gradient clipping utilities (`clip_grad_norm_`, `clip_grad_value_`). When the resolved device type is `"npu"`, pass `foreach=False` to both. Determine `is_npu` once after device resolution and reuse it:

```python
is_npu = device.type == "npu"
```

For example:

```python
# Optimizer constructor
optimizer = optim.AdamW(model.parameters(), ..., foreach=False if is_npu else None)

# Gradient clipping utility
torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm, foreach=False if is_npu else None)
```

This applies to all generated training code in this skill, including `evaluator.py` finetune/train_from_scratch paths.
